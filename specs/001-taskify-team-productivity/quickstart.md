# 빠른 시작 검증 가이드: Taskify 팀 생산성 플랫폼

이 기능이 실제로 동작하는지 엔드투엔드로 검증하기 위한 실행 가이드다.
전체 구현 코드가 아니라, 실행 순서와 기대 결과만 다룬다.

## 사전 준비

- Node.js 20 LTS 설치
- 저장소 클론 후 `backend/`, `frontend/` 각각에서 의존성 설치:
  `npm install`

## 1. 백엔드 준비 및 실행

```bash
cd backend
npx prisma migrate dev      # SQLite 스키마 생성 (data-model.md 기준)
npm run seed                 # 사전 정의된 5명의 사용자 시드 (User 모델)
npm run dev                  # REST API + Socket.IO 서버 기동
```

**기대 결과**: `GET /api/users` 호출 시 PM 1명, Engineer 4명 총 5개의
사용자가 반환된다(contracts/api.md 참고).

## 2. 프런트엔드 실행

```bash
cd frontend
npm run dev
```

브라우저에서 두 개의 창(또는 시크릿 창 포함)으로 앱을 연다 —
실시간 반영(스토리 3)을 두 사용자 관점에서 확인하기 위함이다.

## 3. 시나리오 1 검증 — 프로젝트와 팀 구성

1. 창 A에서 PM 계정을 선택한다.
2. 이름과 설명을 입력해 새 프로젝트를 생성한다.
   - **기대 결과**: 프로젝트 목록에 즉시 나타난다(SC-001, 2분 이내).
3. 프로젝트에 엔지니어 2명을 팀원으로 추가한다.
4. 창 B에서 추가된 엔지니어 중 한 명의 계정을 선택한다.
   - **기대 결과**: 창 B에서도 같은 프로젝트가 보인다.

## 4. 시나리오 2 검증 — 칸반 보드 태스크 관리

1. 창 A에서 프로젝트를 열고 태스크를 하나 생성한다.
   - **기대 결과**: "To Do" 컬럼에 태스크가 나타난다.
2. 태스크를 창 B의 엔지니어에게 할당한다.
   - **기대 결과**: 담당자 이름이 태스크 카드에 표시된다.
3. 태스크를 "To Do"에서 "In Progress"로 드래그 앤 드롭한다.
   - **기대 결과**: 창 A에서 즉시 이동하고, 창 B에서도 새로고침 없이
     5초 이내에 같은 위치로 이동한 것이 보인다(SC-002).
4. (선택) 네트워크를 일시적으로 차단한 뒤 드래그해 실패 케이스를
   재현한다.
   - **기대 결과**: 태스크가 원래 컬럼으로 되돌아가고 에러 메시지가
     표시된다.

## 5. 시나리오 3 검증 — 댓글

1. 창 A와 창 B에서 같은 태스크를 연다.
2. 창 A에서 댓글을 작성해 등록한다.
   - **기대 결과**: 창 B에서 새로고침 없이 5초 이내에 새 댓글이
     나타난다(SC-004).
3. 빈 내용으로 댓글 등록을 시도한다.
   - **기대 결과**: 제출이 거부되고 댓글이 추가되지 않는다(FR-011).

## 6. 품질 게이트 확인

```bash
cd backend && npm run test:coverage
cd frontend && npm run test:coverage
```

- **기대 결과**: 두 워크스페이스 모두 문장/분기/함수/라인 커버리지가
  80% 이상(헌법 원칙 II).

```bash
cd frontend && npm run lint    # eslint-plugin-jsx-a11y 포함
cd frontend && npm run test:a11y   # axe 기반 자동 접근성 검사
```

- **기대 결과**: 접근성 위반 0건(헌법 원칙 III).
